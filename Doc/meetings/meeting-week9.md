## 一、技术栈确认  

根据课程设计文档要求，本子系统采用以下技术方案：

| 技术领域  | 技术选型 | 说明 |
|-------|----------|------|
| 开发平台  | HarmonyOS | 鸿蒙操作系统，课程指定 |
| 开发语言  | ArkTS | 鸿蒙推荐编程语言，基于TypeScript扩展 |
| UI框架  | ArkUI | 声明式UI开发框架 |
| IDE   | DevEco Studio 6.1.0 (Release) | 统一开发环境 |
| 网络通信  | HTTP 协议 | 调用后端知识服务子系统的REST API |
| 本地存储  | Preferences / 关系型数据库 | 用户偏好、历史记录、收藏数据缓存 |
| 语音能力  | HarmonyOS 语音服务 | 语音播讲、语音搜索、语音问答（调用后端问答接口） |
| 以图搜图  | 相机API + 图片选择器 | 拍照/相册上传，特征比对由后端（CLIP+FAISS）完成，移动端负责调用 |
| 音视频播放 | AVPlayer / Video组件 | 文物讲解音视频播放 |
| 图像加载  | Image组件 + 图片缓存 | 高清图片加载与缓存优化 |
| 版本控制  | Git + GitHub | 协作开发 |

> 说明：本子系统不直接操作知识图谱，所有业务数据通过后端 API 获取；以图搜图的特征提取与相似度检索由后端（CLIP+FAISS）完成，移动端仅负责图片采集与结果展示。

## 五、Git 协作规范

为避免代码冲突，全员必须遵守以下规则：

1. **禁止直接推送到 `main` 分支**。
2. 每位组员基于 `main` 创建自己的功能分支，命名格式：`feature/<模块名>`（如 `feature/browse-module`）。
3. 日常开发在自己的分支上进行，每天工作结束前将分支推送至远端做备份。
4. 需要合并到主分支时，在 GitHub 上发起 **Pull Request**，由组长审核后合并。
5. 合并前如遇冲突，由开发者在本地解决冲突后重新推送。

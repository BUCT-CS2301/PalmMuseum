## 四、Git 协作规范
为避免代码冲突，全员必须遵守以下规则：
1. **禁止直接推送到 `main` 分支**。
2. 每位组员基于 `main` 创建自己的功能分支，命名格式：`feature/<模块名>`（如 `feature/browse-module`）。
3. 日常开发在自己的分支上进行，每天工作结束前将分支推送至远端做备份。
4. 需要合并到主分支时，在 GitHub 上发起 **Pull Request**，由组长审核后合并。
5. 合并前如遇冲突，由开发者在本地解决冲突后重新推送。

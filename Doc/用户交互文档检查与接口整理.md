# 用户交互文档检查与接口整理

## 1. 检查结论

当前项目中，用户交互相关内容已经分散写在以下文档中：

- `README.md`
- `Doc/项目管理计划.md`
- `Doc/需求规格说明书.md`
- `Doc/设计文档.md`
- `Doc/第11周开发任务.md`

整体判断：**已有基础说明，但还不算完全完善，和当前代码实现存在几个明显偏差。**

### 1.1 已经覆盖到的内容

现有文档已经覆盖了用户交互模块的主功能范围，包括：

- 点赞
- 收藏
- 评论
- 照片上传
- 评论审核状态
- 个人中心查看收藏、评论、上传内容

其中：

- `Doc/需求规格说明书.md` 已经写到“点赞、收藏及分组管理、评论与回复、用户上传照片”
- `Doc/设计文档.md` 已经写到“点赞收藏、评论回复、照片上传、浏览记录与统计”
- `Doc/设计文档.md` 也已经预留了后续联调接口

### 1.2 还不够完善的地方

对照当前代码实现，文档主要还有下面这些不足：

1. **缺少“评论点赞”说明**
   - 当前代码已经支持给他人评论点赞。
   - 但需求文档和设计文档的接口表里，没有单独列出“评论点赞”接口。

2. **收藏夹分组文档写旧了**
   - 当前代码已经支持：
     - 新建收藏夹
     - 删除收藏夹
     - 将已有收藏移动到其他分组
   - 但设计文档仍偏向“固定分组切换”的描述，没有把“自定义分组增删”写完整。

3. **审核流与当前实现不完全一致**
   - 文档把评论和照片描述成“提交后进入审核中”。
   - 当前前端实现实际上是：先进入 `pending`，约 **1.5 秒** 后自动转为 `approved`。
   - 这个“原型态自动审核”行为没有在文档中写明。

4. **个人主页统计联动未单独写清楚**
   - 当前代码会联动统计：
     - 点赞数
     - 收藏数
     - 评论数
     - 照片数
     - 浏览记录数
   - 但文档对这些统计字段的来源和刷新机制写得比较弱。

5. **接口设计还不够细**
   - 当前文档把点赞/收藏合并成一个 `/user/action`。
   - 这对前后端联调不够清晰，尤其不利于处理：
     - 评论点赞
     - 收藏夹分组 CRUD
     - “我的评论 / 我的上传 / 我的收藏”分页查询
     - 个人主页统计汇总

6. **文档编码存在问题**
   - 当前仓库中的部分中文文档在终端中出现乱码。
   - 这说明文档编码可能不统一，后续建议统一为 `UTF-8`。

- 收藏夹分组查询
- 新建收藏夹
- 删除收藏夹
- 收藏移动分组
- 评论列表查询
- 评论发表
- 评论回复
- 评论点赞
- 用户上传照片
- 上传记录查询
- 审核状态自动流转（原型阶段本地模拟）
- 浏览记录记录与查询
- 个人主页统计汇总

## 4. 建议保留的后端接口清单

下面这份接口清单，是根据当前页面和 `InteractionStore` 的真实调用能力整理出来的。  
如果后续要从本地 `Preferences` 平滑切换到真实后端，建议至少具备这些接口。

## 4.1 文物交互汇总

### 1. 获取文物交互摘要

- 方法：`GET`
- 路径：`/artifacts/{objectId}/interaction-summary`
- 作用：获取当前用户在该文物上的点赞、收藏状态，以及总数

请求参数：

```json
{
  "userId": "string"
}
```

响应示例：

```json
{
  "like": {
    "active": true,
    "count": 28
  },
  "favorite": {
    "active": false,
    "count": 13,
    "groupName": "默认收藏夹"
  }
}
```

---

## 4.2 点赞接口

### 2. 文物点赞

- 方法：`POST`
- 路径：`/artifacts/{objectId}/likes`

请求体：

```json
{
  "userId": "string"
}
```

响应示例：

```json
{
  "active": true,
  "count": 29
}
```

### 3. 取消文物点赞

- 方法：`DELETE`
- 路径：`/artifacts/{objectId}/likes`
- 参数：`userId`

响应示例：

```json
{
  "active": false,
  "count": 28
}
```

## 4.3 收藏与收藏夹接口

### 4. 获取收藏夹分组列表

- 方法：`GET`
- 路径：`/users/{userId}/favorite-groups`

响应示例：

```json
[
  { "name": "默认收藏夹", "count": 8, "isDefault": true },
  { "name": "课堂资料", "count": 3, "isDefault": true },
  { "name": "毕业设计", "count": 5, "isDefault": false }
]
```

### 5. 新建收藏夹

- 方法：`POST`
- 路径：`/users/{userId}/favorite-groups`

请求体：

```json
{
  "groupName": "毕业设计"
}
```

### 6. 删除收藏夹

- 方法：`DELETE`
- 路径：`/users/{userId}/favorite-groups/{groupName}`

说明：

- 删除自定义收藏夹时，建议后端支持把原内容迁移到默认收藏夹

### 7. 获取我的收藏列表

- 方法：`GET`
- 路径：`/users/{userId}/favorites`

查询参数：

```json
{
  "groupName": "默认收藏夹",
  "page": 1,
  "size": 20
}
```

### 8. 新增收藏

- 方法：`POST`
- 路径：`/users/{userId}/favorites`

请求体：

```json
{
  "objectId": "artifact-001",
  "groupName": "默认收藏夹"
}
```

响应示例：

```json
{
  "active": true,
  "count": 14,
  "groupName": "默认收藏夹"
}
```

### 9. 取消收藏

- 方法：`DELETE`
- 路径：`/users/{userId}/favorites/{objectId}`

响应示例：

```json
{
  "active": false,
  "count": 13
}
```

### 10. 移动收藏到其他分组

- 方法：`PATCH`
- 路径：`/users/{userId}/favorites/{objectId}`

请求体：

```json
{
  "groupName": "毕业设计"
}
```

## 4.4 评论接口

### 11. 获取文物评论列表

- 方法：`GET`
- 路径：`/artifacts/{objectId}/comments`

查询参数：

```json
{
  "userId": "string",
  "page": 1,
  "size": 20
}
```

说明：

- 公开区返回已审核评论
- 如果带 `userId`，可额外返回当前用户自己的待审核评论
- 需要支持 `parentId` / `replyTo` 字段，以便前端构建回复树

### 12. 发表评论 / 回复评论

- 方法：`POST`
- 路径：`/artifacts/{objectId}/comments`

请求体：

```json
{
  "userId": "string",
  "content": "这件文物细节很精彩",
  "parentId": "comment-1001",
  "replyTo": "alice"
}
```

响应示例：

```json
{
  "id": "comment-2001",
  "status": "pending",
  "message": "评论已提交，等待审核"
}
```

### 13. 评论点赞

- 方法：`POST`
- 路径：`/comments/{commentId}/likes`

请求体：

```json
{
  "userId": "string"
}
```

响应示例：

```json
{
  "active": true,
  "count": 6
}
```

说明：

- 当前前端实现是“只允许点一次，不支持取消”
- 如果后端希望支持取消，也可以额外提供 `DELETE /comments/{commentId}/likes`

### 14. 获取我的评论

- 方法：`GET`
- 路径：`/users/{userId}/comments`

查询参数：

```json
{
  "page": 1,
  "size": 20
}
```

## 4.5 照片上传接口

### 15. 上传照片

- 方法：`POST`
- 路径：`/artifacts/{objectId}/photos`
- 类型：`multipart/form-data`

表单字段：

- `userId`
- `file`
- `location`
- `description`

响应示例：

```json
{
  "id": "upload-3001",
  "status": "pending",
  "message": "上传成功，等待审核"
}
```

### 16. 获取我的上传照片

- 方法：`GET`
- 路径：`/users/{userId}/photos`

查询参数：

```json
{
  "page": 1,
  "size": 20
}
```

### 17. 查询审核状态

- 方法：`GET`
- 路径：`/audit/status/{contentId}`

响应示例：

```json
{
  "contentId": "upload-3001",
  "contentType": "photo",
  "status": "approved"
}
```

说明：

- 评论和照片都可以共用这类接口

## 5. 最小联调建议

如果后端短期只想先满足当前前端最核心的页面，建议优先实现下面 10 个接口：

1. `GET /artifacts/{objectId}/interaction-summary`
2. `POST /artifacts/{objectId}/likes`
3. `POST /users/{userId}/favorites`
4. `PATCH /users/{userId}/favorites/{objectId}`
5. `GET /users/{userId}/favorite-groups`
6. `GET /artifacts/{objectId}/comments`
7. `POST /artifacts/{objectId}/comments`
8. `POST /comments/{commentId}/likes`
9. `POST /artifacts/{objectId}/photos`
10. `GET /users/{userId}/interaction-stats`

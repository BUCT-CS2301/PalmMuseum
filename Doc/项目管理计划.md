## 1. 项目背景与目标

### 1.1 项目背景
本项目为北京化工大学“软件工程综合实验”课程设计，“海外藏中国文物知识管理与服务平台”的移动端组成部分。本子系统面向手机用户，提供沉浸式的文物浏览、语音导览、以图搜图、用户互动等功能，与知识服务、知识问答、后台管理等子系统协同，形成完整的文物知识服务生态。

### 1.2 子系统目标
基于 HarmonyOS + ArkTS 开发一款功能完整、体验流畅的 App，实现：
- **文物浏览**：首页瀑布流/卡片视图、多维筛选排序、详情页（高清图片、属性、相关推荐）、文物对比。
- **语音导览**：文物语音讲解、语音搜索、语音问答（对接知识问答子系统）。
- **以图搜图**：拍照/相册选取，返回相似文物及相似度。
- **用户交互**：登录注册、点赞收藏、评论留言、上传照片（审核后显示）。
- **个人信息**：个人主页、浏览记录、收藏夹、上传内容管理。

成功标准：通过课程验收，App 可独立运行，并与其他子系统成功集成。

## 2. 技术栈与架构

### 2.1 核心技术选型
| 类别 | 选型 | 说明 |
|------|------|------|
| 开发环境 | DevEco Studio 6.1.0 | 官方 IDE，支持模拟器与真机调试 |
| 语言 | ArkTS | 基于 TypeScript，鸿蒙原生 |
| UI框架 | ArkUI | 声明式 UI，组件化开发 |
| 状态管理 | AppStorage + Preferences | 全局登录状态 + 本地持久化（收藏/评论） |
| 网络请求 | @ohos.net.http | 封装为 HttpApi，后续替换真实接口 |
| 多媒体 | 相机 API、相册选择器、AVPlayer | 以图搜图、语音播放 |
| 语音能力 | @ohos.speech (语音识别) + @ohos.textToSpeech | 语音搜索与讲解 |
| 图像加载 | Image 组件 + 缓存策略 | 支持缩略图、高清图渐进加载 |
| 版本控制 | Git + GitHub | 协作开发 |

### 2.2 系统分层架构
| 层级 | 职责 | 关键模块                                                    |
|------|------|---------------------------------------------------------|
| **展示层** | UI 页面与组件 | Index, HomePage, DetailPage, DiscoverPage, ProfilePage等 |
| **状态层** | 响应式状态管理 | @StorageLink, @State, PreferencesUtil                   |
| **业务逻辑层** | 调用 API、数据处理 | ArtifactService（Mock/真实接口切换）                            |
| **网络层** | 统一请求与错误处理 | HttpApi (GET/POST)                                      |
| **系统能力层** | 调用鸿蒙原生能力 | 相机、相册、语音、音频播放                                           |
| **数据层** | 本地 Mock 数据、持久化 | mockArtifacts, Preferences                              |

## 5. 质量保证与测试策略

### 5.1 质量控制点
- **代码规范**：统一使用 `const`/`let`，禁止 `any`；组件命名 `PascalCase`；每次提交前运行 `Clean Project` 确保无编译错误。
- **UI/UX 一致性**：边距、颜色、字体大小统一使用常量；所有页面包含加载状态、空状态、错误状态。
- **数据可靠性**：Mock 数据与真实接口数据结构保持一致；字段缺失时显示“暂无信息”。
- **性能要求**：列表图片懒加载；详情页大图支持缩放。

### 5.2 测试计划
| 测试类型 | 覆盖功能 | 执行方式 | 通过标准 |
|----------|----------|----------|----------|
| 单元测试 | 工具类（PreferencesUtil、HttpApi） | 模拟调用，验证返回值 | 核心方法正确率 100% |
| 功能测试 | 所有必做功能（按任务分配文档） | 手工测试 + 测试用例记录 | 无阻断性错误，功能符合需求 |
| 兼容性测试 | 模拟器 + 真机 | 运行全部核心流程 | 布局无错位，交互正常 |
| 集成测试 | 调用真实后端接口 | 逐个页面验证数据正确性 | 所有接口返回数据正常展示 |
| 回归测试 | 每次合并 main 前 | 运行核心流程（登录、浏览、搜索） | 原有功能不被破坏 |

### 5.3 缺陷管理
- 使用 GitHub Issues 记录缺陷：标题、复现步骤、优先级（高/中/低）、负责人、状态（待处理/修复中/已关闭）。
- 高优先级缺陷需在 2 天内修复。

## 9. 关键约定

### 9.1 代码仓库与分支
- **仓库地址**：`https://github.com/BUCT-CS2301/PalmMuseum.git`
- **主分支**：`main`（仅通过 PR 合并）
- **功能分支**：`feature/模块名`（如 `feature/browse`、`feature/image-search`）
- **提交信息**：`<类型>: <简短描述>`，如 `feat: 完成首页列表卡片切换`

### 9.2 命名规范
- **页面文件**：`PascalCase`，如 `HomePage.ets`
- **公共类**：`PascalCase`，如 `HttpApi`
- **常量**：`UPPER_SNAKE_CASE`

### 9.3 跨组接口约定
- 调用其他子系统（知识服务、问答）的 API 统一使用 `HttpApi` 封装。
- 接口地址统一配置在 `Config.ets` 中，支持切换测试/生产环境。
